# Node Topology 连线说明文档

## 概述

Node Topology 展示渠道的业务（Business）、节点（Node）、渠道（Channel）、机构（Organization）之间的拓扑关系。连线使用 `leader-line-new` 库的 `pointAnchor` 实现精确锚点定位。

## 拓扑结构

```
Business → Node → Channel → Organization
```

从左到右依次为：业务卡片 → 节点卡片 → 渠道卡片 → 机构卡片。

## 隐藏节点

以下类型的节点不渲染卡片，但连线会穿透到可见的祖先/后代节点，并在连线上显示标签（`#N TYPE`格式）：

| 类型 | 线型 |
|------|------|
| LEASED_LINE | 实线 |
| IPSEC_VPN | 虚线 |
| INTERNET | 虚线 |

隐藏节点在拓扑图下方以图例卡片形式展示：`[#1 节点名称 TYPE]`

## 锚点系统

### 通用 6 点锚点（Node ↔ Node 互联）

```
     1   2   3
   ┌─────────────┐
 4 │             │ 1
   │             │
 5 │             │ 2
   │             │
 6 │             │ 3
   └─────────────┘
     1   2   3
```

- 右侧：1(100%,25%) 2(100%,50%) 3(100%,75%)
- 左侧：4(0%,25%) 5(0%,50%) 6(0%,75%)
- 上侧：(25%,0%) (50%,0%) (75%,0%)
- 下侧：(25%,100%) (50%,100%) (75%,100%)

**分配规则（每边）：**
- 1条连线 → 中间点（2号/5号）
- 2条连线 → 上+下点（1号+3号 / 4号+6号）
- 3条连线 → 全部（1+2+3号）

**方向判断（getSides）：**
- 同列节点：下方目标用 bottom→top，上方目标用 top→bottom
- 跨列节点：右侧目标用 right→left，左侧目标用 left→right

**Gravity 延伸：** 所有锚点向所在边方向延伸 20px。

### Business 出线锚点（右侧 3 点）

```
   ┌─────────────┐
   │             │ 1 (100%,25%)
   │             │
   │             │ 2 (100%,50%)
   │             │
   │             │ 3 (100%,75%)
   └─────────────┘
```

**分配规则：**
- 1条出线 → 2号（中间）
- 2条出线 → 1号 + 3号
- 3条出线 → 1号 + 2号 + 3号

**箭头方向：**
- OUTBOUND：Business → Node（箭头指向 Node）
- INBOUND：Node → Business（箭头指向 Business）

---

### Node 接收 Business 连线锚点（9 点）

```
        2    3    4
   ┌─────────────────┐
 1 │                 │
   │                 │
 0 │                 │
   │                 │
 5 │                 │
   └─────────────────┘
        6    7    8
```

| 编号 | 位置 | 坐标 |
|------|------|------|
| 0 | 左侧中间 | (0%, 50%) |
| 1 | 左侧上方 | (0%, 25%) |
| 2 | 上边左 | (25%, 0%) |
| 3 | 上边中 | (50%, 0%) |
| 4 | 上边右 | (75%, 0%) |
| 5 | 左侧下方 | (0%, 75%) |
| 6 | 下边左 | (25%, 100%) |
| 7 | 下边中 | (50%, 100%) |
| 8 | 下边右 | (75%, 100%) |

**分配规则：**

**1个业务连接：** 使用 0号（左侧中间）

**多个业务连接（按 seq 排序后上下分半）：**

上半部分（seq 较小的业务）：

| 数量 | 锚点顺序 |
|------|----------|
| 1个 | 3 |
| 2个 | 4, 2 |
| 3+个 | 4, 3, 2, 1 循环 |

下半部分（seq 较大的业务，从最大 seq 开始反向分配）：

| 数量 | 锚点顺序 |
|------|----------|
| 1个 | 7 |
| 2个 | 8, 7 |
| 3+个 | 8, 7, 6, 5 循环 |

---

### Channel → Organization 连线锚点（15 点）

```
      u1  u2  u3  u4  u5
   ┌──────────────────────┐
   │                      │ r1
   │                      │
   │                      │ r2
   │                      │
   │                      │ r3
   │                      │
   │                      │ r4
   │                      │
   │                      │ r5
   └──────────────────────┘
      d1  d2  d3  d4  d5
```

| 区域 | 锚点 | 坐标 |
|------|------|------|
| 上边 | u1-u5 | (17%,0%) (33%,0%) (50%,0%) (67%,0%) (83%,0%) |
| 右侧 | r1-r5 | (100%,10%) (100%,30%) (100%,50%) (100%,70%) (100%,90%) |
| 下边 | d1-d5 | (17%,100%) (33%,100%) (50%,100%) (67%,100%) (83%,100%) |

**分配规则：**

**1个机构：** r3（右侧中间）

**多个机构（上半部分）：**

| 数量 | 锚点顺序 |
|------|----------|
| 1个 | r2 |
| 2个 | r1, r2 |
| 3+个 | u1, u2, u3, u4, u5, r1, r2 循环 |

**多个机构（下半部分，从下方往上反向分配）：**

| 数量 | 锚点顺序 |
|------|----------|
| 1个 | r4 |
| 2个 | r5, r4 |
| 3+个 | d1, d2, d3, d4, d5, r5, r4 循环 |

**机构接收端：** 固定左侧中间 (0%, 50%)

## 连线样式

| 属性 | 值 |
|------|-----|
| 路径 | fluid |
| 线宽 | 2px |
| 颜色 | 主题色 `--devui-brand` + CC 透明度 |
| 标签颜色 | 主题色 `--devui-text` |
| Gravity | 20px（沿锚点所在边方向延伸） |
| 虚线 | IPSEC_VPN / INTERNET 类型 |

## 编辑模式

- 点亮编辑按钮后，卡片可点击编辑
- 卡片名称后显示 `S{seq}` 序号标记
- Node Topology 标题旁显示 + 按钮可新增节点
- 隐藏节点图例卡片也可点击编辑
